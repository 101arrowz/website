Dependency ../website/generate-api-docs/example.flow:394

A Dependency denotes a connection between two assets (likely some effect from the importee is expected - be it a side effect or a value is being imported).

Property Descriptions
  • moduleSpecifier: E.g. "lodash" in import {add} from "lodash";

  • isAsync: Whether the environment can load this natively (FIXME).

  • isEntry: Whether this should become a new entry (e.g. no hash).

  • isOptional: Whether the importer might expect an error when resolving failed.

  • isURL: Whether an URL is expected (rather than the language-specific behaviour).

  • isWeak: Whether this dependency does not provide any values for the importer itself.

  • isDeferred: Whether this dependency was never resolved because it was deemed unnecessary/unused (based on symbols).

  • loc: Used for error messages, the code location that caused this dependency.

  • env: the environment of the importee.

  • target: FIXME importer or importee?

  • sourceAssetId: used for error messages, the importer.

  • sourcePath: used for error messages, the importer.

  • symbols: a Map<export name of importee, placeholder in importer>.

  • pipeline: a named pipeline to be chosen (if the moduleSpecifier didn't specify one).

An asset (usually represents one source file).

Property Descriptions
  • The: file system where the source is located.

  • isIsolated: Whether this asset should be put in a separate bundle. VERIFY

  • isInline: Whether this asset will/should later be inserted back into the importer.

  • isSplittable: FIXME

  • isSource: Whether this is asset is part of the users project (and not of an external dependencies).

  • type: Usually corresponds to the file extension

  • symbols: a Map<export name, identifier name;

  • sideEffects: Whether this asset can be omitted if none if it's exports are being used (set by ResolveResult)

  • uniqueKey: In an inline asset, id might not be enough for a unique identification. FIXME

  • astGenerator: The type of the AST.

Method Descriptions
  • getAST: Returns to current AST. See notes in subclasses (Asset, MutableAsset).

  • getCode: Returns to current source code. See notes in MutableAsset.

  • getBuffer: Returns the contents as a buffer.

  • getStream: Returns the contents as a stream.

  • getMap: Returns the sourcemap (if existent).

  • getMapBuffer: A buffer representation of the sourcemap (if existent).

  • getConfig: Used to load config files, (looks in every parent folder until a module root) for the specified filenames. packageKey can be used to also check pkg#[packageKey].

  • getPackage: Returns the package.json this file belongs to.
