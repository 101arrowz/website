Method Descriptions
  • generateEmptyMap: Generates an empty map from the provided fileName and sourceContent

  • addEmptyMap: Generates an empty map from the provided fileName and sourceContent

  • addRawMappings: Appends raw VLQ mappings to the sourcemaps

  • addBufferMappings: Appends a flatbuffer to this sourcemap Note: The flatbuffer buffer should be generated by this library

  • addIndexedMapping: Appends a Mapping object to this sourcemap Note: line numbers start at 1 due to mozilla's source-map library

  • addIndexedMappings: Appends an array of Mapping objects to this sourcemap This is useful when improving performance if a library provides the non-serialised mappings
    Note: This is only faster if they generate the serialised map lazily Note: line numbers start at 1 due to mozilla's source-map library

  • addName: Appends a name to the sourcemap

  • addNames: Appends an array of names to the sourcemap's names array

  • addSource: Appends a source to the sourcemap's sources array

  • addSources: Appends an array of sources to the sourcemap's sources array

  • getSourceIndex: Get the index in the sources array for a certain source file filepath

  • getSource: Get the source file filepath for a certain index of the sources array

  • getNameIndex: Get the index in the names array for a certain name

  • getName: Get the name for a certain index of the names array

  • indexedMappingToStringMapping: Convert a Mapping object that uses indexes for name and source to the actual value of name and source
    Note: This is only used internally, should not be used externally and will probably eventually get handled directly in C++ for improved performance

  • extends: Remaps original positions from this map to the ones in the provided map
    This works by finding the closest generated mapping in the provided map to original mappings of this map and remapping those to be the original mapping of the provided map.

  • getMap: Returns an object with mappings, sources and names This should only be used for tests, debugging and visualising sourcemaps
    Note: This is a fairly slow operation

  • findClosestMapping: Searches through the sourcemap and returns a mapping that is close to the provided generated line and column

  • toBuffer: Returns a flatbuffer that represents this sourcemap, used for caching

  • toVLQ: Returns a serialised map using VLQ Mappings

  • delete: A function that has to be called at the end of the SourceMap's lifecycle to ensure all memory and native bindings get de-allocated

  • stringify: Returns a serialised map
